# カジノゲーム実装 完全分析

## 1. ブラックジャック (Blackjack)

### ゲームの目的
プレイヤーがディーラーと対戦し、手札の合計を21に近づける（超えない）ゲーム。
ディーラーより強い手札を作れば勝ち。

### プレイヤーの操作フロー
```
ベット → ディール → ヒット/スタンド選択 → 結果確認 → もう一度
```

### 各フェーズの詳細

#### フェーズ1: ベット
- **何を見せるか**: チップトレイ（$1/$5/$25/$100/$500）、ベット額表示、ベットサークル
- **プレイヤーの操作**: チップをクリックしてベット額決定 → ディールボタン
- **現在の実装**: ✅ ChipTrayコンポーネントで5段階チップ、✅ BETサークル表示

#### フェーズ2: ディール
- **何を見せるか**: 
  - シュー（茶色いカードボックス、右上）
  - バーンカード（1枚捨てる） ← **未実装**
  - カードが1枚ずつテーブルに配られるアニメーション
  - 正しい配布順: 左から右へプレイヤー→プレイヤー→...→ディーラー（1枚目表向き、2枚目裏向き）
- **現在の実装**: ❌ ディール順序が不正確（D→P→D→P）、❌ バーンなし、✅ カード出現アニメーション

#### フェーズ3: プレイ（ヒット/スタンド）
- **何を見せるか**: 
  - 手札スコア表示
  - ヒットボタン（人差し指を立てる手信号アイコン） ← **未実装**
  - スタンドボタン（手を水平に振る手信号アイコン） ← **未実装**
  - ディーラーのアップカード表示
- **現在の実装**: ✅ スコア表示、✅ ヒット/スタンドボタン、❌ 手信号アニメーションなし

#### フェーズ4: 結果
- **何を見せるか**: 
  - ディーラーのホールカード公開
  - ディーラーのドロー（17以上になるまで）アニメーション ← **未実装**
  - 勝敗表示（WIN/LOSE/PUSH/BJ）
  - ペイアウト計算（例: $10 × 3:2 = $15）
- **現在の実装**: ✅ PayoutBox（計算過程表示）、❌ ディーラードローの1枚ずつ表示なし

1. ディール順序を実際のカジノに合わせる（バーン→プレイヤー→プレイヤー→...→ディーラー）
2. ディーラーのドローを1枚ずつアニメーション表示
3. 手信号アイコン（👆/✋）をより実際の動きに近いアニメーションに

## 2. ルーレット (Roulette)

### ゲームの目的
回転するホイールにボールを落とし、止まった番号を予想するゲーム。
賭けた場所にボールが入れば配当を受け取れる。

### プレイヤーの操作フロー
```
ベット選択 → チップ配置 → スピン → 結果確認
```

### 各フェーズの詳細

#### フェーズ1: ベット選択
- **何を見せるか**: 
  - 数字グリッド（0-36、3列×12行） ← **表示はあるが未実装: スプリット/ストリート/コーナーベットの区別なし**
  - アウトサイドベット（赤/黒/偶数/奇数/1-18/19-36/ダズン）
  - カラードチップ（プレイヤーごとに異なる色のチップ） ← **未実装**
  - レーストラック（コールベット用） ← **未実装**
- **プレイヤーの操作**: ベットエリアをクリック → チップが置かれる
- **現在の実装**: ❌ クリックした番号の区別なし（'straight'固定）、❌ カラードチップなし、❌ レーストラックなし

#### フェーズ2: スピン
- **何を見せるか**: 
  - ホイール回転アニメーション（3秒）
  - ポインターで止まった番号を指示
- **現在の実装**: ✅ ホイールSVG回転、❌ ホイールとベッティングレイアウトの位置関係が不正確

#### フェーズ3: 結果
- **何を見せるか**: 
  - ディーラーの宣告（"Number 17, Black!"） ← **未実装**
  - 該当番号がハイライト
  - 的中ベットの配当計算
  - 外れたベットのチップを回収 ← **未実装**
- **現在の実装**: ✅ PayoutBox、❌ 番号ハイライトなし、❌ チップ回収/配当のアニメーションなし

1. 数字グリッドのクリックでストレートアップ/スプリット/ストリート/コーナーを区別
2. カラードチップシステム（テーブル上のチップがプレイヤーごとに色分け）
3. 結果の番号をグリッド上でハイライト

## 3. クラップス (Craps)

### ゲームの目的
2つのサイコロを振り、出た目の合計に賭けるゲーム。
最も複雑なテーブル構成を持つ。

### プレイヤーの操作フロー
```
ベット → カムアウトロール → ポイント成立 → 追加ベット → ロール続行 → 結果
```

### 各フェーズの詳細

#### フェーズ1: ベット
- **何を見せるか**: Pass Line / Don't Pass / Come / Field のベットエリア ← **大部分未実装**
- **現在の実装**: ❌ Pass Lineのみ、❌ Don't Passなし、❌ Comeなし、❌ Fieldなし、❌ プレイスベットなし

#### フェーズ2: カムアウトロール
- **何を見せるか**: 
  - ダイス振動アニメーション ← ✅
  - スティックマンコール（"Comin' out!" "Yo-leven!" "Snake eyes!"） ← ✅
  - パックをON/OFF切り替え ← ✅
- **現在の実装**: ✅ ダイスアニメーション、✅ スティックマンコール、✅ パック表示

#### フェーズ3: ポイント成立後
- **何を見せるか**: 
  - ポイント表示（例: POINT 6）
  - オッズベット追加 ← **未実装**
  - プレイスベット可能エリア ← **未実装**
- **現在の実装**: ✅ ポイント表示、❌ オッズベットなし、❌ プレイスベットなし

#### フェーズ4: 結果
- **何を見せるか**: 
  - 7 → セブンアウト（負け+シューター交代）
  - ポイント再現（勝ち）
  - ペイアウト計算
- **現在の実装**: ✅ 勝敗表示、✅ PayoutBox

## 4. ポーカー (Poker Texas Hold'em)

### ゲームの目的
2枚の手札＋5枚のコミュニティカードから最良の5枚の役を作り、他のプレイヤーと競う。

### プレイヤーの操作フロー
```
プリフロップ → フロップ → ターン → リバー → ショーダウン
```

### 各フェーズの詳細

#### フェーズ1: プリフロップ
- **何を見せるか**: 
  - 各プレイヤーに2枚のカード配布（1枚ずつ）
  - ディーラーボタン表示 ← **未実装**
  - ブラインドポスト ← **未実装**
  - ベッティングアクション ← **未実装**
- **現在の実装**: ✅ カード配布、❌ ボタンなし、❌ ブラインドなし、❌ ベットなし

#### フェーズ2: フロップ/ターン/リバー
- **何を見せるか**: 
  - バーンカード（1枚捨てる） ← ✅
  - コミュニティカードを1枚ずつめくるアニメーション ← ✅
- **現在の実装**: ✅ バーン表示、✅ カードアニメーション

#### フェーズ3: ショーダウン
- **何を見せるか**: 
  - AIプレイヤーのカード公開 ← ✅
  - 役判定とランク表示 ← ✅（C(7,5)=21通り チェックに修正済み）
  - 勝者表示
- **現在の実装**: ✅ 全員の役表示、✅ 勝者判定

## 5. バカラ (Baccarat)

### ゲームの目的
プレイヤー手札とバンカー手札のどちらが9に近いかを予想する。
タブロー（固定ルール表）に従い3枚目が自動配布される。

### プレイヤーの操作フロー
```
ベット（Player/Banker） → カード配布 → 結果確認
```

### 各フェーズの詳細

#### フェーズ1: ベット
- **何を見せるか**: 
  - Player / Banker / Tie の3つのベットエリア ← **Tie未実装**
  - コミッショントラッカー ← ✅
- **現在の実装**: ✅ Player/Banker選択、✅ コミッション表示、❌ Tieベット直接選択なし

#### フェーズ2: カード配布
- **何を見せるか**: 
  - シューからカード配布 ← ✅
  - バンカー→プレイヤーの順に配布
  - ナチュラル（8/9）の即時判定 ← **未実装**
- **現在の実装**: ✅ カードアニメーション、❌ ナチュラル判定のハイライト表示なし

#### フェーズ3: 結果
- **何を見せるか**: 
  - Player/Banker/Tieの勝者表示
  - スコアボード（Big Road） ← **未実装**
  - コミッション（Banker勝ちの5%） ← ✅
- **現在の実装**: ✅ 勝者表示+コミッション、❌ スコアボードなし

## 1. 確率の基礎

### 1.1 基本公式
- **確率 P** = 望ましい結果の数 / 全結果の数
- **期待値 EV** = Σ(確率 × 結果の価値)
- **ハウスエッジ** = |EV| / ベット額 × 100%

2つの6面サイコロで出る合計値と確率:

|------|------------|------|------|
| 2 | 1 | 2.78% | 1/36 |
| 3 | 2 | 5.56% | 2/36=1/18 |
| 4 | 3 | 8.33% | 3/36=1/12 |
| 5 | 4 | 11.11% | 4/36=1/9 |
| 6 | 5 | 13.89% | 5/36 |
| 7 | 6 | 16.67% | 6/36=1/6 |
| 8 | 5 | 13.89% | 5/36 |
| 9 | 4 | 11.11% | 4/36=1/9 |
| 10 | 3 | 8.33% | 3/36=1/12 |
| 11 | 2 | 5.56% | 2/36=1/18 |
| 12 | 1 | 2.78% | 1/36 |

**合計**: 36通り（すべての組み合わせ）

### 1.3 カード確率
52枚の標準デッキから特定のカードを引く確率:
- 特定のカード1枚: 1/52 = 1.92%
- 特定の値のカード（例: 任意のA）: 4/52 = 1/13 = 7.69%
- 特定のスートのカード: 13/52 = 1/4 = 25%

## 6. ブラックジャック戦略表の数学的根拠

各セルの決定は、**期待値最大化**の原理に基づく。

例: 16 vs 10 の場合
- Standの期待値: 約-0.540（54%の確率で負け）
- Hitの期待値: 約-0.535
- Surrenderの期待値: -0.500
→ Surrenderが最適（できない場合はHit）

例: 11 vs 6 の場合
- Hitの期待値: 約+0.130
- Doubleの期待値: 約+0.260
→ Doubleが最適（倍のベットで倍の期待値）

### 9.2 標準偏差の実務的意味

標準偏差は「1ベットあたりの結果のバラつき」を表す。

**例**: BJ $10ベット、σ=$0.99/ベット（$1ベットあたり）
```
実際のσ = $10 × $0.99 = $9.90
100ハンド後の95%信頼区間:
  = 100 × (-$0.006 × $10) ± 1.96 × $9.90 × √100
  = -$6 ± $194.04
```

**ルーレット ストレートアップ $1ベット**:
```
σ = $5.76
100スピン後の95%信頼区間:
  = 100 × (-$0.053) ± 1.96 × $5.76 × √100
  = -$5.30 ± $112.90
```

## 11. 相互検証（同じ確率を異なる方法で計算）

### 11.1 クラップスパスライン勝率の二重検証

**方法1: 直接確率計算**
```
カムアウト勝ち: 7(6通り) + 11(2通り) = 8/36 = 22.22%
カムアウト負け: 2(1) + 3(2) + 12(1) = 4/36 = 11.11%
ポイント成立: 24/36 = 66.67%

ポイント成立後:
  4→4再現: 3/9 = 33.33%
  5→5再現: 4/10 = 40.00%
  6→6再現: 5/11 = 45.45%
  8→8再現: 5/11 = 45.45%
  9→9再現: 4/10 = 40.00%
  10→10再現: 3/9 = 33.33%

総勝率 = P(即決勝ち) + Σ P(ポイント成立) × P(ポイント再現)
       = 22.22% + (8.33%×33.33% + 11.11%×40.00% + 13.89%×45.45%
                   + 13.89%×45.45% + 11.11%×40.00% + 8.33%×33.33%)
       = 22.22% + (2.78% + 4.44% + 6.31% + 6.31% + 4.44% + 2.78%)
       = 22.22% + 27.07%
       = 49.29%
```

**方法2: モンテカルロシミュレーション**
100万回のシミュレーションによる推定: 49.29-49.31%

**検証結果**: 一致 ✅

## 12. 回帰分析の基礎（カジノへの応用）

### 12.1 線形回帰によるRTP推定
スロットマシンのRTP（還元率）を推定する。

```
n回プレイ後のペイアウト率 = β₀ + β₁ × n
β₀ は初期の偏り、β₁ は実際のRTPへの収束速度
```

### 12.2 ロジスティック回帰によるプレイヤー分類
プレイヤーの行動データからカードカウンターを識別する。

```
P(カウンター) = 1 / (1 + e^(-(β₀ + β₁×bet_size + β₂×bet_variance + β₃×win_rate)))
```

特徴量: ベットサイズの変動、インシュランスの頻度、プレイ速度、席の選択
